# pytest-DSL VSCode 扩展功能详解

## ▶️ 单文件运行与步进调试

- 编辑器标题、右键菜单和文件顶部 CodeLens 可直接运行或调试 `.dsl/.auto` 文件。
- 可在可执行 DSL 步骤上选择“从此处调试”；起点之前的步骤仍会执行，但不会暂停。
- 调试暂停后支持 `F10` 单步、`F5` 继续和 `Shift+F5` 停止，并高亮当前执行行。
- 运行输出、错误输出、关键字跟踪和调试步骤状态统一写入 `pytest-DSL` 输出面板。
- Python 解释器复用项目 `.venv/venv` 与 `pytest-dsl.pythonPath`；启动前会验证 `pytest_dsl.workbench.runner` 确实可用。
- 状态栏 `Config` 选择器自动发现项目 YAML，支持多选、语法校验、项目级记忆和有序配置方案。
- 当前配置统一用于运行、调试、变量补全和定义跳转；“自动”模式保留 pytest-dsl 默认加载规则。
- “使用配置运行/调试…”支持单次临时选择，不覆盖项目默认选择。

配置方案示例：

```json
{
  "pytest-dsl.configProfiles": {
    "local": ["config/base.yaml", "config/local.yaml"],
    "test": ["config/base.yaml", "config/test.yaml"]
  },
  "pytest-dsl.activeConfigProfile": "local"
}
```

## 📊 关键字分类浏览器

### 🎯 核心特性
- **分层分类显示**: 自动按照 `category` 字段分组关键字
- **收藏夹系统**: 个人收藏的常用关键字
- **实时统计**: 显示每个分类的关键字数量
- **图标区分**: 不同类型的关键字使用不同图标

### 🔧 支持的分类类型
- **🔧 内置关键字** (`builtin`): pytest-dsl核心关键字
- **🔍 自定义关键字** (`custom`): 项目特定关键字  
- **📚 库关键字** (`library`): 第三方库关键字
- **👤 用户关键字** (`user`): 用户自定义关键字
- **⭐ 收藏夹** (`favorites`): 用户收藏的关键字

### 🎮 交互功能
- **点击预览**: 点击关键字查看详细信息
- **右键菜单**: 丰富的上下文操作
- **展开/折叠**: 支持分类的展开和折叠
- **拖拽操作**: 未来支持拖拽插入

## 🔍 智能搜索系统

### 🎯 搜索范围
- **关键字名称**: 模糊匹配关键字名称
- **关键字说明**: 搜索文档说明内容
- **参数名称**: 搜索参数名称和描述
- **分类信息**: 搜索分类和来源信息

### ⚡ 搜索特性
- **实时搜索**: 输入即搜索，无需等待
- **模糊匹配**: 支持拼音、缩写等模糊搜索
- **结果排序**: 按匹配度和收藏状态排序
- **结果限制**: 可配置最大搜索结果数量

### 🎨 搜索界面
- **增强显示**: 显示参数数量、来源、说明
- **操作选项**: 插入、预览、编辑、收藏等操作
- **快捷操作**: 支持多种插入方式

## ⭐ 收藏夹管理

### 📋 收藏功能
- **一键收藏**: 右键菜单或内联按钮快速收藏
- **收藏显示**: 收藏的关键字显示星标图标
- **收藏分类**: 收藏夹作为独立分类显示在顶部
- **优先显示**: 插入关键字时收藏项优先显示

### 💾 数据持久化
- **全局存储**: 收藏数据存储在VSCode全局状态中
- **跨项目共享**: 收藏夹在不同项目间共享
- **自动同步**: 收藏状态实时同步到界面

### 🔄 管理操作
- **批量管理**: 支持批量添加/移除收藏
- **导入导出**: 支持收藏夹的导入和导出
- **清理功能**: 自动清理无效的收藏项

## 🎯 智能插入系统

### 📝 插入方式
1. **仅插入名称**: 只插入关键字名称
2. **插入参数模板**: 插入完整的参数结构

### 🔧 参数模板生成
```dsl
HTTP请求[
    客户端=${1:default}
    配置=${2:请求配置}
    会话=${3:会话名称}
    保存响应=${4:响应变量名}
    禁用授权=${5:false}
    模板=${6:模板名称}
    断言重试次数=${7:0}
    断言重试间隔=${8:1}
]
```

### ✨ 智能特性
- **Snippet支持**: 使用VSCode snippet语法
- **Tab跳转**: 支持Tab键在参数间跳转
- **默认值**: 自动填充参数默认值
- **类型提示**: 参数类型和描述提示

## 📋 关键字数据格式

### 🏗️ JSON结构
```json
{
  "summary": {
    "total_count": 99,
    "category_counts": {
      "builtin": 18,
      "custom": 81
    },
    "source_counts": {
      "builtin:pytest-dsl内置": 18,
      "custom:pytest-dsl": 81
    }
  },
  "keywords": [
    {
      "name": "HTTP请求",
      "category": "builtin",
      "source_info": {
        "type": "builtin",
        "name": "pytest-dsl内置",
        "display_name": "pytest-dsl内置",
        "module": "pytest_dsl.keywords.http_keywords",
        "plugin_module": ""
      },
      "parameters": [
        {
          "name": "客户端",
          "mapping": "client",
          "description": "客户端名称，对应YAML变量文件中的客户端配置",
          "default": "default"
        }
      ],
      "documentation": "执行HTTP请求..."
    }
  ]
}
```

### 📊 字段说明
- **name**: 关键字显示名称
- **category**: 分类标识 (`builtin`, `custom`, `library`, `user`)
- **source_info**: 来源信息对象
  - **type**: 类型标识
  - **display_name**: 显示名称
  - **module**: 模块路径
- **parameters**: 参数数组
  - **name**: 参数显示名称
  - **mapping**: 参数映射名称
  - **description**: 参数描述
  - **default**: 默认值
- **documentation**: 详细说明文档

## 🐛 故障排除

### ❗ 常见问题

**1. 关键字浏览器为空**
- 检查关键字JSON文件是否存在
- 确认JSON文件格式正确
- 使用"生成关键字文件"功能

**2. 搜索无结果**
- 检查搜索关键词拼写
- 尝试使用更简单的关键词
- 检查搜索结果数量限制

**3. 收藏夹不显示**
- 确认已收藏关键字
- 检查收藏夹配置选项
- 尝试刷新关键字浏览器

**4. 插入功能异常**
- 确认光标位置在编辑器中
- 检查文件类型是否为.dsl或.auto
- 尝试重新打开文件

### 🔧 调试方法
1. 打开VSCode开发者工具
2. 查看控制台错误信息
3. 检查扩展输出日志
4. 重启VSCode窗口
